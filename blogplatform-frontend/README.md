# Blog Platform Frontend

## API base URL

- В продакшене фронту нужен origin бэкенда. Устанавливай переменную окружения **`VITE_API_BASE`** (без `/api` в конце, axios добавит его сам). Тот же корень используется для SignalR-подключений `/hubs/...`.
- Vite читает `.env*` только во время сборки, поэтому для Netlify `VITE_API_BASE` должен быть задан в окружении сборки (например, через `netlify.toml` или UI). Во время выполнения переменная из `.env` уже не подхватывается.
- Для локальной разработки и тестирования используй `VITE_API_BASE="https://localhost:7141"`, чтобы фронтенд отправлял запросы на нужный бэкенд.
- Если переменная не задана в прод-сборке, приложение продолжит ходить на относительный `/api`, но в консоли появится предупреждение. Чтобы это работало, должен быть настроен прокси.

## Локальная прод-сборка

- Для проверки прод-сборки локально можно использовать файл `.env.production` (см. пример), но помни, что этот файл читается только при `npm run build`.

## Локальный запуск

```bash
npm install
npm run dev
```

## Ручные проверки сессии

1. **Персистентность**: залогиниться, обновить страницу и убедиться, что пользовательские данные подгружаются без повторного ввода пароля.
2. **Выход**: нажать кнопку выхода или инициировать `authService.logout()`, убедиться, что cookie с JWT очищена и переход происходит на `/login`.
